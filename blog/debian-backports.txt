<div class="sectionbody">
<div class="paragraph">
<p>Debian has gained a well deserved reputation of being a robust and reliably system. Part of that comes from the reliance on extensive testing of the packages available from the official repositories. However, these can be a little behind in terms of version numbers which may cause them to lack features or bug fixes. This isn&#8217;t an issue most of the time (latest doesn&#8217;t necessarily mean better), but it can become an issue.</p>
</div>
</div>
</div><menu class="toc"><li class="toc__item"><a href="#_install_from_the_backports_repository">Install From The Backports Repository</a></li><li class="toc__item"><a href="#_install_from_testing_branch">Install From Testing Branch</a></li></menu>
<div class="sect1"><h2 id="_install_from_the_backports_repository">Install From The Backports Repository<a class="anchor" href="#_install_from_the_backports_repository">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>The <em>backports</em> repository offers re-compiled packages using libraries available for the current stable branch, albeit at higher versions of those available in the regular repositories. While these packages haven&#8217;t been tested as thoroughly, they are the best choice for using more up to date software without risking the stability of the system. To install packages from this repository, first you need to tell the system where to download them from, by adding an entry to <code>/etc/apt/sources.list</code>:</p>
</div>
<div class="paragraph">
<p><code>deb <a href="http://deb.debian.org/debian">http://deb.debian.org/debian</a> stable-backports main</code></p>
</div>
<div class="paragraph">
<p>This uses the classic one-line format, and it will work just as well, but you can also use the now standard Deb822 format instead. For that, create a new file in <code>/etc/apt/sources.list.d</code> with a <code>.sources</code> extension and the following contents:</p>

<p>This is unfortunate, since there&#8217;s an outstanding bug within <code>nftables</code> that is already fixed in a later version. This is where the <em>testing</em> branch can really help by offering packages that are at a higher version.</p>
</div>
<div class="paragraph">
<p>Before adding the appropriate repositories to your list, it&#8217;s important that you set a preferences &mdash; also known as pinning down the repository sources &mdash; to avoid installing any packages from the <em>testing</em> accidentally. As mentioned, and as the name suggests, this packages are still undergoing testing and may introduce bugs and other stability issues. You want to make sure that you only install them explicitly.<br>
For that, create a new configuration file in <code>/etc/apt/preferences.d/99-debian-testing</code> &mdash; the name does not matter, but rules specified here are processed in alphabetical order.</p>
</div>

<div class="paragraph">
<p>This ensures that all packages coming from the testing repository will have a priority of 50, which is lower than the default of 500 from the stable repositories. This way, if a package with the same name exists on both repositories, the stable one will take precedence.<br>
The backports repository has a default priority of 100 which is why you don&#8217;t need to do this step when adding the backports repository alone.</p>
