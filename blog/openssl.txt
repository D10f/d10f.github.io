<div class="sect1"><h2 id="_asymmetric_cryptography">Asymmetric Cryptography<a class="anchor" href="#_asymmetric_cryptography">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>Also referred to as public cryptography, asymmetric cryptography uses values or <em>keys</em> that are safe to share publicly over an insecure network while retaining the three core principles of integrity, confidentiality and authentication.</p>
</div>
<div class="sect2"><h3 id="_rsa">RSA<a class="anchor" href="#_rsa">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>Is one of the most popular asymmetric encryption algorithms, named after the last names of their creators, Ron Rivest, Adi Shamir and Leonard Adleman, in 1977. It works by creating a pair of mathematically related <em>keys</em>, such that one key is used to encrypt a message while the other can decrypt it.</p>
</div>
<div class="paragraph">
<p>To generate a new key-pair:</p>
</div>

<div class="paragraph">
<p>By default, <code>openssl genrsa</code> already produces the keys of 2048 bit length, but you could specify this value at the end of the command. There&#8217;s a configuration file that can be modified to update this default value &mdash; as well as other settings &mdash; found at <code>/etc/openssl/openssl.conf</code>.</p>
</div>

            <div class="admonition admonition--left warning">
                <svg xmlns="http://www.w3.org/2000/svg">
                    <use href="/sprite.svg#icon-warning" />
                </svg>
                <p>As of this writing, 2048-bit key lengths is still considered reasonably secure. However, it's already projected that this won't be enough by the year 2030.</p>
            </div>
<div class="paragraph">
<p>This produces a PEM-formatted file, which is a base64-encoded representation of the binary data that makes up the key. One advantage of this format is how easy it is to work with, generally, particularly for things like copy/paste.</p>
</div>
<div class="paragraph">
<p>If we inspect the contents of this file:</p>
</div>

        </div>
<div class="paragraph">
<p>The <code>-in</code> argument takes the file to read, <code>-text</code> produces a human-readable, plain text representation and <code>-noout</code> prevents additional output in the original, base64-encoded format to be included. This command will output the various components involved in the RSA keys. The <strong>modulus</strong> and <strong>publicExponent</strong> combined are the values used for the <em>public</em> key, while the <strong>privateExponent</strong> and the other values are used for the <em>private key</em>.</p>
</div>
</div>
<div class="sect2"><h3 id="_elliptic_curve">Elliptic Curve<a class="anchor" href="#_elliptic_curve">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>Elliptic curve cryptography follows the same principle as RSA of using a key-pair, but the mathematics behind it are different. In essence, it&#8217;s a more efficient way of doing the same thing, allowing for smaller key sizes while retaining the same level of security.</p>
</div>

            <div class="admonition admonition--right important">
                <svg xmlns="http://www.w3.org/2000/svg">
                    <use href="/sprite.svg#icon-important" />
                </svg>
                <p>Security is a relative term. Something that is considered secure by today's standards may not be so in the next 10 years. In addition, security needs to be used within a context, and balanced with convenience. We could use massive key sizes all the time but that would make user experience needlessly detrimental, due to the long computation times.</p>
            </div>
