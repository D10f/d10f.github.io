<div class="paragraph">
<p>The <code>-name</code> and <code>-iname</code> options allow specifying the name of the file you&#8217;re looking for. Since an exact match is required, using glob patterns can be really convenient to find multiple files that follow a similar structure. You can also use <code>-regex</code> and <code>-iregex</code> instead, to take advantage of full regular expressions in your query.</p>

<div class="sect1"><h2 id="_bfs">BFS<a class="anchor" href="#_bfs">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>This is an interesting one that can be quite handy to know about. To all intents and purposes it&#8217;s identical to <code>find</code> in how it&#8217;s used and what it does, but differs in <em>how</em> it does it. Whereas <code>find</code> uses a <em>depth-first search</em> algorithm, <code>bfs</code> uses a <em>breath-first search</em> one, hence the name.</p>
</div>
<div class="paragraph">
<p>In summary, both of these utility tools will <em>eventually</em> find the file(s) in question. It&#8217;s a matter of how choosing the better tool depending on what you are looking for, and whether you have some idea of where that might be found.</p>
</div>
<div class="paragraph">
<p>Checkout the <a href="https://github.com/tavianator/bfs">project&#8217;s repository</a> to learn more about how to install it and what other options are available.</p>
</div>
</div>
</div>
<div class="sect1"><h2 id="_locate">Locate<a class="anchor" href="#_locate">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>The <code>locate</code> command does not search for files directly, it searches an index that is created by the command <code>updatedb</code> which automatically runs once a day. This in turn means that results are usually very fast with the downside that the index may not be entirely up-to-date. While this is updated automatically for you, you can still run <code>sudo updatedb</code> manually.</p>
</div>
<div class="paragraph">
<p>The results are based on a match on the file name, and it returns them as a list by their absolute path. It won&#8217;t however return any matches within directories without read permissions such as <code>/etc</code> or <code>/bin</code>.</p>
</div>
</div>
</div>
