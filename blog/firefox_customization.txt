<p>While this article is largely aimed at Firefox, the same steps should work for any forks of the project such as LibreWolf. To avoid repeating myself here, assume that every time I mention Firefox I&#8217;m also referring to any derived browsers as well.</p>

<div class="sect1"><h2 id="_prerequisites">Prerequisites<a class="anchor" href="#_prerequisites">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="sect2"><h3 id="_the_chrome_directory">The "chrome" directory<a class="anchor" href="#_the_chrome_directory">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>

            <div class="admonition admonition--left tip">
                <svg xmlns="http://www.w3.org/2000/svg">
                    <use href="/sprite.svg#icon-tip" />
                </svg>
                <p>The term "chrome" refers to any visible aspect of a browser aside from the webpages themselves. It appears this used to be a slang automotive enthusiasts would use to refer customizing their cars with shiny features. It has no connection to Google Chrome, which was named ironically so as a pun with the aim to create a minimal, distraction-free experience for their users.</p>
            </div>
<div class="paragraph">
<p>When Firefox launches, it will read the current profile in use and load things like custom preferences, extensions and of course, the topic of this article, styling rules. These styles are read from a CSS file located at <code>chrome/userChrome.css</code>. Make sure to create this location (both the directory and the file) if it doesn&#8217;t already exist (see Appendix A).</p>
</div>
</div>
<div class="sect2"><h3 id="_enable_the_browser_toolbox_mode">Enable The Browser Toolbox Mode<a class="anchor" href="#_enable_the_browser_toolbox_mode">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>This is a two-step process:</p>
</div>
<div class="olist arabic">
<ol class="arabic">
<li>
<p>Enable the option "Enable remote debugging" in the developers tools. Press <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <code>I</code> and move on to the "Settings", under the menu at the right-hand side.</p>
</li>
<li>
<p>Navigate to <code>about:config</code> up at the address bar, and set the option "toolkit.legacyUserProfileCustomizations.stylesheets" to true.</p>
</li>
</ol>
</div>

            <div class="admonition admonition--right important">
                <svg xmlns="http://www.w3.org/2000/svg">
                    <use href="/sprite.svg#icon-important" />
                </svg>
                <p>In LibreWolf, you will find this setting available in the Settings > LibreWolf > "Allow userChrome.css customization".</p>
            </div>
</div>
</div>
</div>
<div class="sect1"><h2 id="_inspecting_the_browser">Inspecting The Browser<a class="anchor" href="#_inspecting_the_browser">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>Even if you already know CSS, how do you know what selectors to use? That&#8217;s when inspecting the browser&#8217;s <strong>chrome</strong> can be helpful. To access the toolbox, press <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Alt</kbd> + <code>I</code> and accept the incoming network request (should be from localhost) at the prompt.</p>
</div>

            <div class="admonition admonition--left caution">
                <svg xmlns="http://www.w3.org/2000/svg">
                    <use href="/sprite.svg#icon-caution" />
                </svg>
                <p>This can be a vector for attack which is why it's disabled by default. It's important that you don't accept any prompts like this if you didn't explicitly request them.</p>
            </div>
<div class="paragraph">
<p>To inspect elements freely, click on the element picker &mdash; you can focus it directly with the keyboard shortcut <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>C</kbd> &mdash; and hover your mouse over any of the elements of Firefox. This should highlight them also in the inspector, the box describing the markup of each element. On the right, you can start manipulating elements in real time by entering CSS rules, and see the changes take effect until you like what you see.</p>
</div>
<div class="paragraph">
<p>For example, let&#8217;s place the new tab button all the way to right of the browser toolbar, and remove the all tabs button.</p>
</div>

          <figure class="imageblock">
            <picture>
              <img width="undefined" src="/firefox_custom_toolbar_before.jpg" alt="Before making changes in the browser inspector" loading="lazy" />
              <figcaption>Figure 1. Before making changes in the browser inspector</figcaption>
            </picture>
          </figure>
        
<div class="paragraph">
<p>This seems easy enough, change the display property to <strong>none</strong> on the all tabs button, and create a margin of <strong>auto</strong> on the new tab button.</p>
</div>

          <figure class="imageblock">
            <picture>
              <img width="undefined" src="/firefox_custom_toolbar_after.jpg" alt="After making changes in the browser inspector" loading="lazy" />
              <figcaption>Figure 2. After making changes in the browser inspector</figcaption>
            </picture>
          </figure>
        
<div class="paragraph">
<p>Once you are happy with the changes you&#8217;ve made, move on to the "Changes" tab (still within the inspector view). You will see there all the changes that you&#8217;ve made and copy the entire rules, which you can then paste inside that <code>userChrome.css</code> file that you created earlier. Keep in mind that changes will only take effect after you restart the browser.</p>
</div>

            <div class="admonition admonition--right tip">
                <svg xmlns="http://www.w3.org/2000/svg">
                    <use href="/sprite.svg#icon-tip" />
                </svg>
                <p>If you are new to CSS, take your time and play around. There's no rush.</p>
            </div>
<div class="paragraph">
<p>One word of caution about this: the "Changes" tab will show you all the changes you&#8217;ve made to the chrome, including the ones that you may not want to include in your final CSS. I suggest to work slowly, making small changes at a time. Close the browser inspector and re-open it again to have a clean slate for the next change that you have in mind.</p>
</div>

          <figure class="imageblock">
            <picture>
              <img width="undefined" src="/firefox_custom_toolbar_css_changes.jpg" alt="Changes made to the chrome" loading="lazy" />
              <figcaption>Figure 3. Changes made to the chrome</figcaption>
            </picture>
          </figure>
        
</div>
</div>
<div class="sect1"><h2 id="_organizing_your_code">Organizing Your Code<a class="anchor" href="#_organizing_your_code">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>While you could write all of your CSS rules inside a single file, it can get out of hand very quickly. It&#8217;s probably fine for a few rules but if you have multiple, you might want to create a dedicated file and use the <code>@import</code> CSS directive to keep things modular.</p>
</div>
<div class="paragraph">
<p>For example, you may want to create a file called <code>toolbar.css</code> and import it from <code>userChrome.css</code>:</p>
