<div class="sectionbody">
<div class="paragraph">
<p>A shell is program that provides an interface to the operating system, by processing input in the form of text. The prompt is the part of the shell that disappears while a command is being processed, or running, and comes back when it&#8217;s ready to accept more input. This post explores how this works and how to customize it to fit your needs.</p>
</div>

<div class="sect1"><h2 id="_different_types_of_shells">Different Types Of Shells<a class="anchor" href="#_different_types_of_shells">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>Over the decades there have been many types of shell interpreters that provide different sets of features. Many of the concepts are similar but the syntax of built-in capabilities change.</p>
</div>
<div class="paragraph">
<p>The rest of this post assumes <a href="https://www.gnu.org/software/bash/manual/html_node/Controlling-the-Prompt.html">Bash</a>, as being one of the most popular and widely implemented in most Linux distributions by default.</p>
</div>
</div>
</div>

</div>

<div class="sect1"><h2 id="_using_font_icons">Using Font Icons<a class="anchor" href="#_using_font_icons">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>Now that we know how to use different colors and output whatever information we like with functions, let&#8217;s spice things up with icons. Since most icons are simply text, all we need is to download and setup our terminal emulator of choice to use a font compatible with icons.</p>
</div>
<div class="paragraph">
<p><a href="https://www.nerdfonts.com/">Nerd Fonts</a> is a collection of fonts that support this, and includes popular fonts that have been "patched" with this capability. Download whatever font(s) you want to <code>~/.local/share/fonts</code>. This should location will be picked up by your terminal emulator and other programs whenever you want to change the font that program uses. Check your the documentation of your distribution to know more about how fonts should be setup if this doesn&#8217;t work. To test if things are working, open a new terminal window and copy/paste one of the icons from their cheatsheet. If it shows up correctly, we can proceed.</p>
</div>
<div class="paragraph">
<p>At this point you can directly copy/paste whatever icons you want from the cheat-sheet to be part of the prompt. Another approach that can be useful for a more "programatic" approach, or simply to remember the exact name of the icon for future reference, is to use the <a href="https://github.com/ryanoasis/nerd-fonts/wiki/Icon-Names-in-Shell">shell script provided by Nerd Fonts</a>. This makes the same icons available as variables.</p>
</div>

This will allow me to write the prompt almost in a much more readable way.</p>

<p>Remember, the <code>tput</code> function needs to be escaped when used in the prompt or Bash will lose track of the cursor position. But now, the escaping happens inside each function that prints it&#8217;s own color escape sequence. Because of this there&#8217;s another escape sequence that Bash needs to use for non-printable characters: <code>001</code> and <code>002</code>.</p>

<p>Also it&#8217;s important, in this example, to keep track of the spaces. We don&#8217;t want our functions to output emtpy spaces when there are no background jobs to print, or we&#8217;re not inside a directory tracked by git. This is why they are all written back to back without spaces, and they all output one extra space at the end when needed.</p>
