<div class="sect1"><h2 id="_problem_statement">Problem Statement<a class="anchor" href="#_problem_statement">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>You need to install an operating system on your computer (the client), but you don&#8217;t have any external media device available to install from (maybe because the physical ports are damaged).</p>
</div>
<div class="paragraph">
<p>On the same network you have another computer (the server) with the ISO file for the operating system you want to install. You want to use this computer to act as a server, using the ISO file stored there to boot over the local network.</p>
</div>
<div class="paragraph">
<p>You will need to install software on this machine to turn it into a server capable of listening for network requests to set up the connection between it, and the client.</p>
</div>

            <div class="admonition admonition--left important">
                <svg xmlns="http://www.w3.org/2000/svg">
                    <use href="/sprite.svg#icon-important" />
                </svg>
                <p>There are multiple ways to set this up. This post attempts to provide a straight forward, practical approach following a step-by-step instruction. Please keep this in mind when comparing and contrasting the information you see here with other sources.</p>
            </div>
</div>
</div>
<div class="sect1"><h2 id="_definitions">Definitions<a class="anchor" href="#_definitions">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h2>
<div class="sectionbody">
<div class="paragraph">
<p>These are the parts involved in this process. They are defined in the context of Pre-Execution Environment (PXE), booting a computer over the network.</p>
</div>
<div class="sect2"><h3 id="_dhcp_server">DHCP Server<a class="anchor" href="#_dhcp_server">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>The Dynamic Host Configuration Protocol is used to assign communication parameters to each host on a network, most commonly an IP address for identification purposes. In this case however, we&#8217;re interested in other parameters that will help with the net-boot process, by means of directing the client machine to the location of the TFTP server.</p>
</div>
</div>
<div class="sect2"><h3 id="_tftp_server">TFTP Server<a class="anchor" href="#_tftp_server">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>The Trivial File Transfer Protocol is used for transferring files over the network, commonly used in the early stages of the boot process due to its simplicity compared to more sophisticated alternatives. It will be used to serve the client with the bootloader and other files as needed.</p>
</div>
</div>
<div class="sect2"><h3 id="_network_bootloader">Network Bootloader<a class="anchor" href="#_network_bootloader">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>A bootloader is the software that runs before the operating system. It&#8217;s responsible from loading and transferring control over to the <em>kernel</em> i.e., Linux, which in turn loads the operating system itself.<br>
Since we want to boot over the network we need a PXE bootloader that will be served from the TFTP server.</p>
</div>
</div>
<div class="sect2"><h3 id="_grub">GRUB<a class="anchor" href="#_grub">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>GNU GRUB is another bootloader specifically made for personal computers. Most Linux distributions today use it as a mean to select the operating system to boot into (even when there is only one). We will use this once all files have been downloaded over the network and our system is ready to boot into the remote ISO image, which will be downloaded separately over HTTP.</p>
</div>
</div>
<div class="sect2"><h3 id="_http_server">HTTP Server<a class="anchor" href="#_http_server">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>Once the client has received the bootloader files, has loaded the GRUB menu and selected an option to choose from (for simplicity, in this example there will be only one option), the ISO image needs to be downloaded by the client. We&#8217;ll use a basic but reliable server to that effect, using the well-known HTTP protocol.</p>
</div>
</div>
<div class="sect2"><h3 id="_iso_file">ISO File<a class="anchor" href="#_iso_file">
                    <svg aria-hidden="true">
                        <use href="/sprite.svg#icon-link" />
                    </svg>
                </a></h3>
<div class="paragraph">
<p>The target image that we want to boot into. For example, we&#8217;ll go with Linux Mint 21.3 "Virginia".</p>
</div>
</div>
</div>
</div>
